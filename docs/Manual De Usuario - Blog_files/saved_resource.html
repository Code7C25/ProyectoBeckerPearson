Inicio de sesión y registro

Acceso (login)/abrirsesion.php

Ruta típica: login.html o abrirsesion.php. Introducí usuario y contraseña y hacé click en Ingresar.

  1. Ir a /login.html.
  2. Completar usuario y contraseña.
  3. Si falla, revisá conexion.php y la tabla de usuarios en la base de datos.
Registro de nuevos usuarios/nuevousuario.php

Formulario disponible en nuevousuario.html. Campos típicos: nombre, email, contraseña y perfil.

Crear, editar y eliminar posts

El flujo general para publicar contenido:

  1. Entrar con una cuenta con permisos.
  2. Ir a nuevopost.php para crear un post nuevo.
  3. Para editar usar editar.php o editarus.php según el caso.
  4. Eliminar con eliminar.php o el icono de tacho en la interfaz.

Campos importantes

  • Título — string
  • Contenido — HTML o texto

Administración de usuarios

Archivos relevantes: adm_usuario.php, adm_usuarios.php, crearusuario.php, editarus.php, eliminar.php.

Crear usuario

  1. Ir a crearusuario.php o usar el formulario nuevousuario.html.

Editar usuario

Usar editarus.php. Validar que la contraseña sea hasheada.

Imágenes y perfiles

Las imágenes se almacenan en carpetas como /imagenes o /Perfiles. Verificá permisos 755/775 al subir archivos desde PHP.

  • Controlá el tipo MIME y el tamaño antes de mover el archivo.
  • Evitar nombres de archivo con espacios; usar uniqid() o un hash.

Estructura de archivos (resumen)

Listado corto de los archivos más relevantes encontrados en el ZIP:

  • index.php — página principal
  • abrirsesion.php — login
  • crearusuario.php, nuevousuario.php — registro
  • nuevopost.php, guardarnuevopost.php — posts
  • conexion.php — datos de conexión a la BD
  • Carpeta /Perfiles y /imagenes — assets

Despliegue y requisitos

  • PHP 7.4+ recomendado (ajustar según funciones usadas).
  • Servidor web (Apache / Nginx) con mod_php o PHP-FPM.
  • Base de datos MySQL / MariaDB; revisar conexion.php.
  • Permisos de carpetas: asegurar que la carpeta de uploads sea escribible por PHP.

Pasos rápidos

  1. Subir archivos al servidor en la carpeta pública.
  2. Crear la base de datos e importar el esquema (si tenés .sql).
  3. Editar conexion.php con credenciales.
  4. Probar /index.php y login.

Seguridad y buenas prácticas

  • Usar password_hash() y password_verify() para contraseñas.
  • Preparar sentencias (prepared statements) para evitar SQL injection.
  • Validar y sanear todo input de usuario (HTML/JS) para evitar XSS.
  • Limitar tamaño y tipo de archivos subidos.
  • Deshabilitar display_errors en producción.

Solución de problemas comunes

Error de conexión a la DB

Revisá conexion.php, usuario, contraseña, host y que el servicio de MySQL esté corriendo.

Imágenes que no suben

Revisá permisos y upload_max_filesize / post_max_size en php.ini.

Login fallido

Verificar hashing y que la consulta SELECT devuelve el usuario esperado.

Preguntas frecuentes (FAQ)

  1. ¿Puedo cambiar el diseño? Sí: editá CSS o reemplazá plantillas en la carpeta /BLOG/BLOG.
  2. ¿Dónde están las imágenes de perfil? /Perfiles dentro del ZIP.
  3. ¿Cómo hago una copia de seguridad? Exportá la base de datos y guardá la carpeta de uploads.
