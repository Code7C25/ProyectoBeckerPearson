# **1. Descripción General del Proyecto**

El *Paparazzi Project* es un sistema creado para mejorar la comunicación, organización y acceso a información dentro de un entorno educativo o institucional. La idea principal es ofrecer una plataforma simple, intuitiva y rápida para que los usuarios puedan obtener datos relevantes sin depender de procesos manuales o confusos.

Este proyecto nace de la necesidad de hacer más eficiente la forma en que estudiantes, docentes o personal reciben información y se relacionan con los recursos disponibles.



# **2. Problema Detectado**

Antes de este proyecto, existían varios problemas:

* **Falta de centralización:** La información estaba dispersa entre mensajes, papeles, grupos y plataformas diferentes.
* **Pérdida de tiempo:** Los usuarios tardaban en encontrar lo que necesitaban.
* **Errores por mala comunicación:** Se repetían dudas porque no había un sistema claro.
* **Poca accesibilidad:** No todos sabían dónde buscar o cómo utilizar la información disponible.


# **3. Solución Propuesta**

El *Paparazzi Project* organiza todo en un solo lugar. Brinda:

* Un **acceso rápido** a la información.
* Una **interfaz clara** para que cualquier usuario pueda manejarse sin complicaciones.
* Un **sistema seguro y confiable**, que permite que todos usen la plataforma sin riesgos ni confusiones.
* Un diseño visual atractivo que ayuda a entender y navegar el sistema con facilidad.


# **4. Tecnologías Utilizadas**


* **Frontend:** HTML, CSS, JavaScript
* **Backend:** PHP 
* **Base de datos:** MySQL
* **Control de versiones:** GitHub
* **Diseño gráfico:** Canva 
* **Frameworks opcionales:** ChatGPT


# **5. Cómo Usar el Sistema**


1. **Ingresar al sistema:** El usuario entra desde el navegador mediante la URL del proyecto.
2. **Seleccionar la sección deseada:** El menú principal muestra las opciones disponibles.
3. **Navegar por la información:** Todo está organizado de forma clara, permitiendo encontrar rápidamente lo que se busca.
4. **Enviar o recibir datos:** Dependiendo del rol, el usuario puede consultar o cargar información.
5. **Finalizar sesión:** Para mantener la seguridad de los datos, se recomienda cerrar sesión al terminar.
